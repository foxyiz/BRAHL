BRAHL-001 – Inconsistent Pass Count Reporting
Severity: High / Critical as noted in summary
Issue Description
Analyze displays varying counts such as 2, 33, and 71 passes across runs, while execution summaries show different totals.
Expected vs Actual Result
Expected: All modules display identical finalized execution counts.Actual: Different modules display different totals.
Recommended Action
Standardize data sources, validate file paths, improve audit tracking, and ensure all modules use the same execution dataset.

BRAHL-002 – zDash Accessibility Failure
Severity: High / Critical as noted in summary
Issue Description
After successful execution, clicking View zDash returns 'File not found' even though results exist and are visible elsewhere.
Expected vs Actual Result
Expected: Generated zDash opens directly from Run module.Actual: File not found error is returned.
Recommended Action
Standardize data sources, validate file paths, improve audit tracking, and ensure all modules use the same execution dataset.

BRAHL-003 – Loop Failure Recovery Tracking
Severity: High / Critical as noted in summary
Issue Description
Two plans failed after tag modifications. Loop → Failures recovered the failures and Verify-Full passed all plans, but recovery history is not clearly presented.
Expected vs Actual Result
Expected: Recovery status should be visible in final reporting.Actual: Recovery workflow lacks clear traceability.

Standardize data sources, validate file paths, improve audit tracking, and ensure all modules use the same execution dataset.
